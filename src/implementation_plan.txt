# 🚀 NestJS Billing Application Requirements

## Project Overview

Build a production-ready Billing/POS Backend using NestJS and MongoDB (Mongoose).

The system should allow administrators and cashiers to manage categories, subcategories, products, carts, and billing operations.

---

* NestJS (Latest Version)
* MongoDB
* Mongoose
* TypeScript
* JWT Authentication
* Swagger Documentation
* Class Validator
* ConfigModule
* Pagination

* Generic CRUD Architecture
* Role Based Access Control

# Category Module

A category represents a top-level grouping of products.

Examples:

* Beverages
* Food
* Desserts
* Snacks

## Schema

```ts
{
  _id: ObjectId,
  name: string,
  description?: string,
  isActive: boolean,
  deletedAt?: Date,
  createdAt: Date,
  updatedAt: Date
}
```

## Endpoints

```http
POST   /categories
GET    /categories
GET    /categories/:id
PATCH  /categories/:id
DELETE /categories/:id
```

## Features

* Search
* Pagination
* Soft Delete

# Cart Module

The cart stores temporary billing data before checkout.

## Schema

```ts
{
  _id: ObjectId,
  cashierId: ObjectId,

  items: [
    {
      productId: ObjectId,
      productName: string,
      quantity: number,
      unitPrice: number,
      totalPrice: number
    }
  ],

  subtotal: number,
  tax: number,
  grandTotal: number,

  createdAt: Date,
  updatedAt: Date
}
```

## Endpoints

```http
POST   /cart/add-item
PATCH  /cart/update-quantity
DELETE /cart/remove-item/:productId
DELETE /cart/clear
GET    /cart
```

## Business Rules

### Add Item

```text
Coffee × 2 = ₹40
Tea × 1 = ₹15
```

### Auto Calculation

```text
Subtotal = Sum(item totals)

Tax = Configurable GST

Grand Total = Subtotal + Tax
```

# Order / Billing Module

When checkout occurs:

1. Create Order
2. Generate Bill Number
3. Store Order
4. Clear Cart

## Schema

```ts
{
  _id: ObjectId,

  orderNumber: string,

  cashierId: ObjectId,

  items: [
    {
      productId: ObjectId,
      productName: string,
      quantity: number,
      unitPrice: number,
      totalPrice: number
    }
  ],

  subtotal: number,
  tax: number,
  discount: number,
  grandTotal: number,

  paymentMethod: string,

  orderStatus: string,

  createdAt: Date
}
```

---

## Order Number Format

```text
ORD-20260530-0001
ORD-20260530-0002
ORD-20260530-0003
```

---

## Checkout Endpoint

```http
POST /orders/checkout
```

### Request

```json
{
  "paymentMethod": "CASH",
  "discount": 20
}
```

### Response

```json
{
  "orderNumber": "ORD-20260530-0001",
  "subtotal": 500,
  "tax": 25,
  "discount": 20,
  "grandTotal": 505
}
```
